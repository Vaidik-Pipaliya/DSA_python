# Custom Agent Rules for DSA Python Repository

This file defines formatting guidelines for documenting LeetCode/DSA problems in this repository.

## Markdown Formatting for Problem Explanations

When writing or editing markdown files (`README.md` or other explanatory markdown files) for DSA problems:

1. **Main Headings (Approach levels)**:
   - Use a **single hash** (`#`) for main approach headings.
   - Examples:
     - `# Brute Force`
     - `# Better Solution`
     - `# Optimal Solution`

2. **Sub-headings (Approach details)**:
   - Use a **triple hash** (`###`) for sub-headings under each approach.
   - Examples:
     - `### Problem Explanation`
     - `### Intuition`
     - `### Algorithm`
     - `### Dry Run`
     - `### Complexity Analysis`
     - `### Code Implementation`
